The WSO2 Application Manager 1.0.0 Released!

The WSO2 Application Manager team is pleased to announce the release of version 1.0.0 of the Open Source Application Manager.

WSO2 Application Manager is a platform for creating, managing, consuming and monitoring Applications. It employs proven SOA best practices to solve a wide range of Application management challenges such as Application provisioning, Application governance, Application security and Application monitoring and Single Sign On. It combines some of the most powerful and mature components of the WSO2's state-of-the-art Carbon platform to deliver a smooth and end-to-end Application management experience while catering to both Application publisher and Application consumer requirements.

WSO2 Application Manager is comprised of several modules.

  • Application Provider: Define new Applications and manage them
  • Application Store: Browse published Applications and subscribe to them
  • Application Gateway: The underlying Application runtime based on WSO2 ESB

WSO2 Application Manager is developed on top of the revolutionary WSO2 Carbon platform (Middleware a' la carte), an OSGi based framework that provides seamless modularity to your SOA via componentization. This release also contains many new features and a range of optional components (add-ons) that can be installed to customize the behavior of the Application Manager. Further, any existing features of the product which are not required in your environment can be easily removed using the underlying provisioning framework of Carbon. In brief, WSO2 Application Manager can be fully customized and tailored to meet your exact Application management needs.

For more information on WSO2 Application Manager please visit http://wso2.com/products/app-manager. Also take a look at the online product documentation.

How to Run

  1. Extract the downloaded zip
  2. Go to the bin directory in the extracted folder
  3. Run the wso2server.sh or wso2server.bat as appropriate
  4. Launch a web browser and navigate to https://localhost:9443/publisher to access the Application provider webapp
  5. Navigate to https://localhost:9443/store to access the Application store
  6. Use "admin", "admin" as the username and password to login as an admin
  7. If you need to start the OSGi console with the server use the property -DosgiConsole when starting the server. The INSTALL.txt file found on the installation directory will give you a comprehensive set of options and properties that can be passed into the startup script

Key Features of WSO2 APP Manager

Following is a categorization of the core features supported by WSO2 APP Manager based on the target user group.

  • Create a Store of all Available Applications:
    • Graphical experience similar to Android Marketplace or Apple App Store.
    • Browse Applications by provider, tags or name.
    • Self-registration to user community to subscribe to Applications.
    • Subscribe to Applications and manage subscriptions on per-application basis.
    • Subscriptions can be at different service tiers based on expected usage levels.
    • Role based access to Application Store; manage public and private Applications.
    • Manage subscriptions at a per-user level.
    • Browse Application documentation, download helpers for easy consumption.
    • Comment on and rate Applications.
    • Forum for discussing Applications usage issues (Available soon in future version).
    • Internationalization (i18n) support.
  • Publishing and Governing API Use:
    • Publish APIs to external consumers and partners, as well as internal users.
    • Supports publishing multiple protocols including SOAP, REST, JSON and XML style services as APIs.
    • Manage API versions and deployment status by version.
    • Govern the API lifecycle (publish, deprecate, retire).
    • Attach documentation (files, external URLs) to APIs.
    • Apply Security policies to APIs (authentication, authorization).
    • Associate API available to system defined service tiers.
    • Provision and Manage API keys.
    • Track consumers per API.
    • One-click deployment to API Gateway for immediate publishing.
  • Route API Traffic:
    • Supports API authentication with OAuth2.
    • Extremely high performance pass-through message routing with sub-millisecond latency.
    • Enforce rate limiting and throttling policies for APIs by consumer.
    • Horizontally scalable with easy deployment into cluster using proven routing infrastructure.
    • Scales to millions of publishers/users.
    • Capture all statistics and push to pluggable analytics system.
    • Configure API routing policies with capabilities of WSO2 Enterprise Service Bus.
    • Powered by WSO2 Enterprise Service Bus.
  • Manage User Community:
    • Self-sign up for Application consumption.
    • Manage user account including password reset.
    • User(Consumer) interaction with Applications via comments and ratings.
    • Support for user communication via forums (Available soon in future version).
    • Powered by WSO2 Identity Server.
  • Govern Complete Application Lifecycle:
    • Manage Application lifecycle from cradle to grave: create, publish, block, deprecate and retire.
    • Publish Applications to partner networks such as ProgrammableWeb (Available soon in future version).
    • Powered by WSO2 Governance Registry.
  • Monitor Application Usage and Performance:
    • All Application usage published to pluggable analytics framework.
    • Out of the box support for WSO2 Business Activity Monitor and Google Analytics.
    • View metrics by user, Application and more.
    • Customized reporting via plugging reporting engines.
    • Monitor SLA compliance.
    • Powered by WSO2 Business Activity Monitor.
  • Pluggable, Extensible and Themeable:
    • All components are highly customizable thru styling, theming and open source code.
    • Storefront implemented with Jaggery (jaggeryjs.org) for easy customization.
    • Pluggable to third party analytics systems and billing systems (Available soon in future version).
    • Pluggable to existing user stores including via JDBC and LDAP.
    • Components usable separately - storefront can be used to front APIs gatewayed via third party gateways such as Intel Expressway Service Gateway.
    • Support for Single Sign On (SSO) using SAML 2.0 for easy integration with existing web apps
  • Easily Deployable in Enterprise Setting:
    • Role based access control for managing users and their authorization levels.
    • Store front can be deployed in DMZ for external access with Publisher inside the firewall for private control.
    • Different user stores for developer focused store-front and internal operations in publisher.
    • Integrates with enterprise identity systems including LDAP and Microsoft Active Directory.
    • Gateway can be deployed in DMZ with controlled access to WSO2 Identity Server (for authentication/authorization) and governance database behind firewall.
  • Support for creating multi-tenanted Applications
    • Run a single instance and provide Application Management to multiple customers
    • Share Applications between different departments in a large enterprise
  • Publishing and Governing Application Use
    • Document an Application using Swagger
    • Restrict Application Access tokens to domains/IPs
    • Ability to block a subscription and restricting a complete application
    • Ability to revoke access tokens
    • Separate validity period configuration for Application Access Token
    • OAuth2 Authorization Code Grant Type Support
    • Configuring execution point of mediation extensions
  • Monitor Application Usage and Performance
    • Improved dashboard for monitoring usage statistics (Filtering data for a date range, More visually appealing widgets)
